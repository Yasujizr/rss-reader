          About proxy urls

          Some RSS feeds are setup so that when you click to view the full article, you must first visit an intermediate URL controlled by the feed's provider, which then redirects you to the article. Content providers sometimes do this to track your reading habits. You can avoid this behavior by providing the technical details on how the extension can bypass the intermediate link. To do this, you need to examine the link to the article (e.g. right click on an article's title and select Copy link location and then paste it somewhere like NotePad). There are two interesting parts to a proxy URL. The first is the base part, that is essentially the link to the intermediate location. This is the first part of the url up to the point of the first ?. The first part is referred to below as the proxy host. The second part is located somewhere after the ?, in the list of pairs of key names and key values. Look for a key name like "url", followed by an = character, followed by what looks like a website address. For example, given the URL http://www.evilproxy.com/itrackurclicks?url=http://www.example.com/example-article.html, the base is http://www.evilproxy.com/itrackurclicks, and the key name of interest is url.
